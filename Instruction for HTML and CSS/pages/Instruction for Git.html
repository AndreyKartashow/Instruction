Основные сведения

Git - это распределенная система контроля версий, используемая для управления изменениями в исходном коде и других файлах проекта. Она была разработана Линусом Торвальдсом для управления разработкой ядра Linux, но с тех пор стала популярным инструментом для совместной работы над проектами в различных отраслях.

Git работает, создавая локальную копию проекта на компьютере пользователя и предоставляя ему множество инструментов для управления версиями файлов. Он позволяет создавать ветки (branches) для разработки новых функций или исправлений ошибок, и объединять их с основной веткой (master) при необходимости. Это позволяет не только сохранять предыдущие версии проекта, но и совместно работать над проектом с другими участниками, даже если они находятся в разных частях мира.

Git использует хэширование для идентификации каждого изменения, что делает его очень надежным и безопасным. Это также позволяет восстановить любую предыдущую версию проекта в любой момент времени.

Git имеет множество инструментов командной строки, но также имеет графические пользовательские интерфейсы, которые делают его более доступным для новичков. Git также поддерживает различные сервисы хостинга репозиториев, такие как GitHub и GitLab.

Перед началом работы с git в VS Code, необходимо выполнить следующие действия:

  • Активировать Terminal (Terminale-New Terminale);
  • ввести в Terminal команду

    "git version"

    (в Terminal отобразиться рабочая версия);
  • ввести в Terminal команду

    "git status"

    (в Terminal отобразиться текущее состояние).

Активация учетной записи, создание файла и репозитория в git

  • Активировать учетную запись командами в консоли

    "git config --global user.name_Ваше имя англ. буквами"

    "git config --global user.email_Ваша почта";

  • создать папку нашего репозитория (File-Open folder-Created folder);
  • в меню "Explorer" появиться наш репозиторий по имени, которым мы его назвали. Далее, создаем новый файл в нашем репозитории (New file), задаем ему название и разрешение.

Сохранение файла и создание первого коммита

После создания файла необходимо совершить фиксацию версии (создать первый коммит), для этого необходимо выполнить следующие действия:

  • Сохраняем наш файл командой горячих клавиш "Ctrl+s";
  • Далее, в Terminal вводим команду

    "git add_название файла"

    (для ввода имени файла достаточно ввести первые буквы название файла и произвести табуляцию клавишей "Tab") и поддтверждаем ее нажатием клавиши "Enter";
  • Вводим в Terminal команду

    "git commit --m "Ваш комментарий""

    и подтвержаем команду нажатием клавиши "Enter".

Работа с коммитами

При необходимости перехода из одного коммита в другой (переход между версиями) нужно использовать следующие команды:

  • отображение всех коммитов за жизненый период файла

    "git log"

  • разница между текущим и крайним коммитом

    "git diff"

  • переход к необходимому коммиту

    "git checkout "имя коммита""

Работа с ветками

Ветки позволяют делить файл на разные подфайлы (подуровни), в которых можно решать отдельные задачи, а затем их переносить в отдельный файл.

Для того чтобы работать с ветками, необходимо знать такие команды, как:

  • создание новой ветки

    "git branch "Название_ветки""

  • слияние веток

    "git merge "Название_ветки""

  • удаление ветки

    "git branch --d "Название_ветки""

  • переход на другую ветку

    "git checkout "Название_ветки""

  • отображение коммитов на разных ветвлениях

    "git log --graph"

Добавление локального репозитория на свой удаленный сервер GitHub

GitHub - это веб-сервис для хостинга и управления репозиториями кода. Он позволяет разработчикам работать над проектами совместно, вносить изменения в код, отслеживать ошибки и совершенствовать проекты. GitHub позволяет создавать репозитории как на своих компьютерах, так и на удаленных серверах, а также предоставляет возможность совместной работы с другими разработчиками через функции контроля версий и запросов на включение изменений. GitHub также широко используется для хранения открытых исходных кодов и ведения блогов о программировании.

  • создать репозиторий в своем аккаунте на GitHub;
  • копировать необходимые команды из GitHub, которые появляются сразу же на главном экране, после регистрации репозитория ;
  • войти в git, активировать Terminal, и вставить скопированные команды из GitHub;
  • После подтверждения команд, Git свяжется с сервисом GitHub и будет знать где находится удаленный сервер.

Копирование чужого удаленного репозитория на локальный компьютер

  • копировать ссылку чужого репозитория с GitHub ;
  • войти в git, активировать Terminal, и вставить скопированные команды из GitHub через команду

    git clone "ссылка"

  • далее необходимо активировать чужой репозиторий на своем компьютере командой

    cd "Название папки"

Связь локального и удаленного репозитория

  • для того чтобы добавить изменения репозитория с локального компьютера на удаленный сервер необходимо использовать команду

    git push

  • для того чтобы обновить данные локального репозитория и подтянуть данные с удаленного сервера, необходимо использовать команду

    git pull

    Важно отметить, что данная команда является составной и имеет такую функцию как сливать файлы

Работа в OpenSours проектах

для работы в открытых проектах используется такая функция как Fork (вилка). Она позволяет копировать чужой репозиторий в свой удаленный и работать с ним. Важно отметить, что работу необходимо выполнять только в своей ветке. Изменения в проекте предлагаются с помощью PullRequest
