Instruction for HTML

Основные сведения

HTML - язык разметки web-сайтов. Он предназначен для формирования структуры сайта и деления его на логические блоки. HTML не является языком программирования.
Для формирования определенной структыры используются теги, которые имеют следующий вид <Наименование тега>. Выделяют как открывающие, так и закрывающие теги. Теги могут быть как двусторонними (полностью оборачивать содержимое), так и одностороними (работать только на открытие).
Важно знать, что структуры в HTML бывают как строчные - представляются в виде строки, так и блочные - которые представленны в виде отдельных блоков и всегда начинаются с новой строки.


Работа с текстом

Основной частью контента является текстовая информация, поэтому важно уметь ее структированно предоставлять пользователю. Ниже предоставленны некоторые теги, которые необходимо знать каждому IT-инженеру для разработки web-сайтов:

  • Для формирования заголовков необходимо использовать теги 'h1' ... 'h6';

  • Для создания горизонтальной линии необходимо использовать тег 'hr';

  • Для формирования параграфа необходимо использовать тег 'p';

  • Для выделения текста жирным шрифтом необходимо использовать тег 'strong';

  • Для выделения текста курсивом необходимо использовать тег 'em';

  • Для разделения абзацев пустой строкой необходимо использовать тег 'br';


Формирование списков

Списки бывают как нумерованные, так и ненумерованные. Они могут использоваться как для предоставления тезисной информации, так и для выделения основных аспектов контента или предоставлять последовательность действий в инструкции.

Для создания блока ненумерованного списка необходимо использовать тег 'ul', затем внести в него тезисы, обернутые тегом 'li'. В результате получится следующий список:

  • клубника;
  • малина;
  • земляника.

Для создания блока нумерованного списка необходимо использовать тег 'ol', затем внести в него тезисы, обернутые тегом 'li'. В результате получим следующий список:

  1. Нарежьте клубнику, малину и землянику;
  2. Нарезанные кусочки фруктов поместите в тарелку;
  3. Заправьте фруктовый салат йогуртом.

Для изменения способа нумерации, используем атрибут type в теге 'ol'

  1. Пушкин был поэтом "Золотого века";
  2. Пушкин был художником и прославился картиной "Девятый вал";
  3. Пушкин был другом Есенина;
  4. Пушкин никогда не учавствовал в дуелях.

Для быстрого создания нескольких тезисов в списке, можно использовать такую конструкцию 'ul>li*n', где n - число необходимых тезисов.


Добавление ссылок

С помощью ссылок осуществляются переходы на другие страницы или сайты. Ссылкой можно перенаправить пользователя на другой ресурс, который к примеру раскрывает отдельные темы и дает более полную информацию.

Для того что бы добавить ссылку на страницу, необходимо использовать тег 'a' с атрибутом 'href=', после которого указывается URL - адрес странички, на которую хотите перенаправить пользователя.

Важно знать, что при таком синтаксисе, ссылка будет открываться в той же вкладке, которую использует пользователь. Для того что бы ссылка открывалась в новой вкладке, необходимо использовать атрибут target="_blank".

Важно так же отметить тот факт, что ссылками можно осуществлять и переход между своими страницами (если страницы html находятся в одной сборке, то после атрибута 'href' мы указываем имя файла).

Добавление изображений

Чтобы сделать страницу сайта более красивой и более информативной, можно в нее добавить изображение или картинку.
Изображение можно добавить как по URL (если изображение находится на сервере), так и по его расположению/наименованию (если изображение храниться на локальном компьютере или в одной сборке с проектом).

Для добавления картинки по URL необходимо использовать тег 'img' с атрибутом 'src', после которого указывается URL - адрес изображения.

Для добавления картинки которая расположена на локальном компьютере или находится в одной сборке с проектом, необходимо использовать тот же тег 'img' с атрибутом 'src', после которого указывается путь/наименование файла.


Добавление таблиц

HTML так же позволяет создавать таблицы, в которые можно вносить информацию. У любой таблицы можно выделить следующую структуру:

  • наименование (шапка таблицы с именованными атрибутами);

  • содержание (поля с информацией по каждому элементу);

  • итоги (поля со сводной информацией по атрибутам элементов).

Для создания блока таблицы необходимо использовать тег 'table', в который последовательно вкладываются подблоки наименование (тег 'thead'), содержание (тег 'tbody') и итоги (тег 'tfoot'), которые в свою очередь заполняются строками (тег 'tr') со столбцами (тег 'th' - для подблока наименование и тег 'td' - для подблока содержание).

Создание форм

HTML так же позволяет создавать разнообразные формы. Формы могут быть представленны в виде полей для ввода опреденных даных с соответствующим типом информации. Например, можно выделить такие формы как:

  • форма для ввода текстовой информации (имя, фамилия);

  • форма для ввода цифровой информации (возраст);

  • форма для ввода даты и времени;

  • форма для ввода многострочного сообщения;

  • форма для установки значения с помощью ползунка;

  • форма для установки значения из предложенных;

  • форма для создания кнопки и другие;

Для создания блока формы в первую очередь необходимо использовать тег 'form', в который затем укладывается подблок тегом 'div'. Для любой формы можно выделить такие параметры как "Наименование формы", который вводится с помощью тега 'label' и "Поле", который вводится с помощью тега 'input' и которое характеризуется такими атрибутами как "тип данных" ('type'), "валидация" ('min/max'), "начальное значение" ('value'), "подсказка" ('placeholder'), "только для чтения" ('readonly') и другие.


Рассмотрим формы для ввода однострочной информации, например имени пользователя, его возраста электронного адреса или пароля.

В остальных полях вводить информацию разрешено. Так же следует обратить внимание на "подсказку", которая исчезает после начала ввода данных. Так же следует обратить внимание на поле "возраст", в котором можно устанавливать значения стрелочками по первоначальной валидации, и обратить на анонимность веденых данных в поле "пароль".

Так же важно знать, что все действия со стороны пользователя (получение или отправка данных на сервер) как правило обрабатывает логика, написанная на языке программирования.
